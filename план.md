## 🎯 ПРО ПРОЕКТ

### Назва

**Аналітичний портал для навчальних закладів з прогнозом успішності студентів**

### Мета

Створення веб-аналітичної системи, яка:

- 📚 Зберігає навчальні дані (студенти, викладачі, предмети, оцінки)
- 📈 Показує поточну успішність та динаміку
- 🔍 Надає базову аналітику
- 🤖 Робить прогноз успішності студентів (ML-алгоритм)
- 👥 Адаптується під різні ролі користувачів (Адмін, Викладач, Студент)
- 📥📤 Підтримує імпорт/експорт даних

### Ключові відмінності від звичайного журналу

✅ Аналітика (середні значення, мін/макс, розподіл)  
✅ Прогнозування (ML-light алгоритм)  
✅ Класифікація ризиків (Safe / At Risk / Critical)  
✅ Динаміка змін та тренди  
✅ What-if аналіз

## ✅ ПОТОЧНИЙ СТАН (feat/auth)

#### 1. Архітектура проекту

- ✅ **Hexagonal Architecture** (Domain → Application → Infrastructure/Presentation)

## 🎁 РОДЗИНКИ ПРОЕКТУ

### 1️⃣ ML-прогноз успішності (ГОЛОВНА РОДЗИНКА)

**Що виглядає як ML, але просто в реалізації:**

#### Алгоритм:

```typescript
class PredictionService {
  predict(grades: Grade[]): PredictionResult {
    const avgRecent = this.calcAvg(grades.slice(-5)); // останні 5
    const avgAll = this.calcAvg(grades);
    const trend = this.calcTrend(grades);

    const predicted = 0.6 * avgRecent + 0.3 * avgAll + 0.1 * trend;

    return {
      predictedScore: Math.min(100, Math.max(0, predicted)),
      confidence: this.calcConfidence(grades.length),
      riskLevel: this.classifyRisk(predicted, trend),
    };
  }

  classifyRisk(score: number, trend: number): RiskLevel {
    if (score >= 75 && trend >= 0) return 'SAFE';
    if (score >= 60 && score < 75) return 'AT_RISK';
    return 'CRITICAL';
  }
}
```

#### Як це подати на захисті:


**Реалізація:**

```typescript
async importStudents(file: Express.Multer.File) {
  const records = await this.parseCsv(file.buffer);
  const preview = records.slice(0, 5); // preview

  // Валідація
  for (const record of records) {
    await this.validateStudentRecord(record);
  }

  // Масове створення
  const students = await this.userRepository.createMany(records);

  return {
    imported: students.length,
    preview,
    errors: []
  };
}
```

### 3️⃣ What-If Analysis (БОНУСНА РОДЗИНКА)

**Студент може запитати:**

> "Якщо я отримаю 95 на іспиті, яка буде моя фінальна оцінка?"

```typescript
async whatIf(studentId: string, subjectId: string, examScore: number) {
  const grades = await this.gradeRepository.findByStudent(studentId, subjectId);
  const currentAvg = this.calcAvg(grades);

  // Додаємо гіпотетичну оцінку
  const hypotheticalGrades = [...grades, { score: examScore, type: 'EXAM' }];
  const predictedAvg = this.calcAvg(hypotheticalGrades);

  return {
    currentAvg,
    predictedAvg,
    improvement: predictedAvg - currentAvg,
    newRiskLevel: this.classifyRisk(predictedAvg)
  };
}
```

**На захисті:**

> "Система підтримує сценарний аналіз (what-if analysis), що дозволяє студентам планувати стратегію навчання на основі прогнозованих результатів."

## 📝 ФОРМУЛЮВАННЯ ДЛЯ ПОЯСНЮВАЛЬНОЇ ЗАПИСКИ

### Розділ "Функціональні можливості":

> Система забезпечує:
>
> 1. Автентифікацію користувачів на основі JWT-токенів
> 2. Розмежування прав доступу (RBAC) для ролей адміністратор, викладач, студент
> 3. Управління навчальними даними (студенти, викладачі, предмети, оцінки)
> 4. Імпорт та експорт даних у форматі CSV для інтеграції з існуючими системами
> 5. Аналітику успішності студентів з розрахунком середніх оцінок, трендів та статистики
> 6. **Прогнозування фінальної успішності** на основі вагової моделі з врахуванням історичних даних
> 7. **Класифікацію студентів за рівнем ризику** академічної неуспішності
> 8. Сценарний аналіз (what-if) для планування навчальної стратегії
> 9. Персоналізовані дашборди для кожної ролі користувача

### Розділ "Технічні рішення":

> Для забезпечення масштабованості та підтримуваності застосовано **Hexagonal Architecture** (Ports & Adapters), що дозволяє ізолювати бізнес-логіку від деталей реалізації інфраструктури. Використання TypeORM забезпечує незалежність від конкретної СУБД. JWT-автентифікація та глобальні guards гарантують безпеку API.

### Розділ "Алгоритм прогнозування":

> Для прогнозування успішності застосовується **вагова модель з аналізом трендів**, яка враховує:
>
> - Середню оцінку за останні N робіт (вага 0.6)
> - Загальну середню оцінку (вага 0.3)
> - Тренд зміни оцінок (вага 0.1)
>
> Класифікація ризику здійснюється на основі порогових значень та динаміки змін, що дозволяє виявляти студентів, які потребують додаткової уваги викладачів.
